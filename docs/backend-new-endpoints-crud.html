Safety Incidents

Full CRUD

Track and manage workplace safety incidents. Supports incident types (vehicle, chemical, slip-fall, equipment, ergonomic), severity levels (minor, moderate, serious), and resolution tracking.

GET /safety-incidents

Retrieve all safety incidents with optional filtering.

Query Parameters

ParameterTypeDescription
site_idstring (UUID)Filter by site
typestringFilter by type: vehicle, chemical, slip-fall, equipment, ergonomic
severitystringFilter by severity: minor, moderate, serious
resolvedbooleanFilter by resolution status (true/false)
date_fromstring (date)Filter incidents on or after date (YYYY-MM-DD)
date_tostring (date)Filter incidents on or before date (YYYY-MM-DD)
pageintegerPage number (default: 1)
limitintegerResults per page (default: 100, max: 500)
 
// Response
{
  "safety_incidents": [
    {
      "id": "uuid",
      "date": "2026-03-15",
      "type": "vehicle",
      "severity": "minor",
      "resolved": false,
      "site_id": "uuid",
      "description": "Minor fender bender in parking lot",
      "root_cause": "Poor visibility",
      "corrective_action": "Added mirrors",
      "reported_by": "John Smith"
    }
  ],
  "pagination": { "page": 1, "limit": 100, "total": 15, "pages": 1 }
}
200 OK 401 Unauthorized

Customer Surveys

Full CRUD

Track customer satisfaction metrics including CSAT scores (1-5), NPS ratings (0-10), First Contact Resolution (FCR), response times, and complaint tracking.

GET /customer-surveys

Retrieve all customer surveys with optional filtering. Includes client name from joined customers table.

Query Parameters

ParameterTypeDescription
client_idstring (UUID)Filter by client/customer
date_fromstring (date)Filter surveys on or after date
date_tostring (date)Filter surveys on or before date
has_complaintbooleanFilter by complaint status
pageintegerPage number (default: 1)
limitintegerResults per page (default: 100, max: 500)
 
// Response
{
  "customer_surveys": [
    {
      "id": "uuid",
      "client_id": "uuid",
      "client_name": "Acme Corporation",
      "date": "2026-03-18",
      "csat": 4.5,
      "nps": 9,
      "fcr_resolved": true,
      "response_time_hrs": 2.5,
      "has_complaint": false,
      "complaint_category": null,
      "feedback": "Great service!"
    }
  ],
  "pagination": { "page": 1, "limit": 100, "total": 45, "pages": 1 }
}

Platform User Activity

New Endpoint

Track user activity on the platform for analytics purposes. Supports page views and action events. User ID is automatically extracted from JWT token.

POST /platform-user-activity/track

Track a user activity event (page view or action). User ID is automatically extracted from the JWT token.

Request Body

FieldTypeRequiredDescription
event_typestringRequiredType of event: "page_view" or "action"
pagestringConditionalPage URL/path (required for page_view events)
actionstringConditionalAction name (required for action events)
metadataobjectOptionalAdditional metadata as key-value pairs
timestampstring (ISO 8601)OptionalEvent timestamp (defaults to current time)
 
// Page View Event
{
  "event_type": "page_view",
  "page": "/dashboard",
  "metadata": {
    "referrer": "/login",
    "tab": "overview"
  }
}

// Action Event
{
  "event_type": "action",
  "action": "export_report",
  "metadata": {
    "report_type": "shipments",
    "format": "csv"
  },
  "timestamp": "2026-03-19T10:30:00Z"
}

// Response
{ "message": "Event tracked" }
201 Created 400 Bad Request

Note: This endpoint is designed to be non-blocking. Even if an error occurs internally, it will return 201 to avoid disrupting the user experience.
