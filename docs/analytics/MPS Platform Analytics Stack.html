Why Microsoft — Not Google Analytics?
MPS IT infrastructure runs on Microsoft Azure. GA4 requires Google Cloud, which enterprise IT departments frequently block. All tools here are native to the Azure ecosystem and integrate directly with existing Power BI dashboards — no cross-cloud data transfer required.
Tool Breakdown
5 Microsoft Tools — What Each Does
Each tool has a specific role. Together they form a complete analytics layer — behavioural tracking, performance monitoring, reporting, data warehousing, and real-time processing.
Microsoft Clarity
User Behavior Analytics
Free
Visual session recording and heatmap tool. Shows exactly how real users interact with the MPS platform — where they click, scroll, and get confused.
  • Session recordings — replay exact user journeys
  • Heatmaps — see most/least clicked areas
  • Rage click detection — identify UI frustration points
  • Dead click detection — broken buttons or links
  • AI-powered insights — auto-summarizes patterns
  • GDPR / CCPA compliant — auto-masks all PII
  • Works on internal enterprise apps
  • Power BI integration available
  • Cannot track API response times
  • Cannot track server-side errors
  • Cannot do custom event analytics
Azure Application Insights
Application Telemetry & KPI Tracking
Pay / GB
The core analytics engine. Tracks every meaningful user action — shipment submissions, filter usage, page views, feature adoption — and monitors system performance.
  • Custom event tracking — any user action becomes a tracked event
  • Active users (DAU / MAU) — daily and monthly counts
  • Funnel analysis — drop-off rates at each step
  • User flows and journey maps
  • Retention and cohort analysis
  • API response time and error rate monitoring
  • Failed request tracking — catches broken pipelines
  • Power BI compatible — direct export to dashboards
  • Near real-time data (seconds, not 30-min GA4 delay)
  • Cannot show heatmaps or session replays
  • Cannot track marketing channels or UTM sources
Microsoft Power BI
Reporting & Visualization
Licensed
The presentation layer. Transforms raw telemetry data from Application Insights into clean, interactive dashboards that leadership can read without opening Azure Portal.
  • Native connector to App Insights
  • Embeddable in the MPS platform UI
  • Scheduled refresh — dashboards auto-update
  • Row-level security — control who sees what
  • Drag-and-drop report builder (Joseph's workflow)
  • Not a tracking tool — only displays data
  • Cannot detect user sessions directly
Azure Monitor
Infrastructure & Health Monitoring
Included
The infrastructure watchdog. Monitors health of all Azure services — databases, APIs, servers — and triggers alerts when something breaks. Prevents broken-demo situations.
  • Uptime and availability monitoring
  • Alert rules — notify team when API is down
  • Metrics dashboards for CPU, memory, query performance
  • Application Insights runs on top of Azure Monitor
  • Not a user-facing analytics tool
  • Cannot track user behavior or KPIs
Azure Synapse Analytics
Data Warehouse — Long-Term Storage
Pay / Query
The long-term memory of the analytics stack. Stores all historical KPI metrics and user telemetry. Enables trend analysis over months and years.
  • Massive-scale data storage
  • SQL-compatible queries on historical data
  • Direct connector to Power BI
  • Combines KPI data + shipment data + user telemetry
  • Phase 2 priority — not needed immediately
  • Requires data engineering setup time

Implementation Guide
5-Step Setup — Priority Order
Steps 1 and 2 can be completed within hours. Steps 3–5 follow as the platform matures.
1
Install Microsoft Clarity — 30 Minutes · Free
Add the Clarity JavaScript snippet to the MPS platform HTML head. Immediately starts recording sessions and generating heatmaps. No backend changes required.
<script>
  (function(c,l,a,r,i,t,y){
    c[a]=c[a]||function(){(c[a].q=c[a].q||[]).push(arguments)};
    t=l.createElement(r);t.async=1;t.src="https://www.clarity.ms/tag/"+i;
    y=l.getElementsByTagName(r)[0];y.parentNode.insertBefore(t,y);
  })(window, document, "clarity", "script", "YOUR_PROJECT_ID");
</script>
2
Add Azure Application Insights SDK — 1–2 Hours
Install the App Insights SDK. Once connected, page views and sessions track automatically. Then add custom events for shipment submission, report usage, and filter clicks.
// npm install @microsoft/applicationinsights-web

import { ApplicationInsights } from '@microsoft/applicationinsights-web';
const appInsights = new ApplicationInsights({
  config: { connectionString: 'YOUR_CONNECTION_STRING' }
});
appInsights.loadAppInsights();
appInsights.trackPageView();

// Track shipment submission:
appInsights.trackEvent({ name: 'shipment_submitted' });
3
Connect App Insights → Power BI — 2–4 Hours
Use the Azure Application Insights connector in Power BI Desktop. Build the 7th dashboard — Platform Analytics — showing all 6 Domain 05 KPIs. Set scheduled refresh to update every hour.
4
Configure Azure Monitor Alerts — 1 Hour
Set alert rules: API response time > 3 seconds, failed request rate > 5%, availability < 99%. Notifies engineering via email or Microsoft Teams before issues reach users.
5
Azure Synapse for Long-Term Storage — Phase 2
After 3–6 months of data, set up Azure Synapse for historical KPI storage at scale. Enables year-over-year trends and contract renewal evidence. Not required for the immediate demo.
