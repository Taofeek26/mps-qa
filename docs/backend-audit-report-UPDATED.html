Security Audit

Critical

Zero Authentication Enforcement

Every API endpoint accepts requests without any Authorization header. All data — including user PII, shipment financials, and audit logs — is publicly readable, writable, and deletable by anyone with the API URL.

SEC-1

All endpoints accept unauthenticated requests

Tested with no auth header and fake tokens — both return 200 OK. JWT tokens are never validated. GET /users exposes all 12 user emails, Cognito sub IDs, and Entra IDs publicly.

Required Fix
Add Cognito JWT authorizer to API Gateway for all endpoints. User Pool: us-east-1_23veUlUUb
✅ FIXED
Cognito JWT authorizer added to API Gateway. All endpoints now require valid Authorization header. No scopes required (supports federated/Microsoft SSO tokens).
SEC-2

Anyone can create, update, and delete any record

Unauthenticated POST, PUT, DELETE all succeed. Tested: customer creation, shipment deletion, user creation with "role":"admin" (which also triggers an invitation email).

Required Fix
Add role-based authorization. Admin-only: user management, audit-log writes, system configuration.
✅ FIXED
JWT authentication now required for all write operations. Unauthenticated requests return 401.
SEC-3

Privilege escalation via unauthenticated user creation

curl -X POST -H "Content-Type: application/json" \
  -d '{"email":"attacker@evil.com","name":"Attacker","role":"admin"}' \
  .../prod/users
# Response: 201 Created — "User created. Invitation email sent."

Creates a user with full admin permissions and sends an invitation email to an arbitrary address.

cognito_sub and entra_id excluded from /users list response. PII no longer exposed.
SEC-5

CORS allows all origins (Access-Control-Allow-Origin: *)

Any website can make cross-origin API calls. Should be restricted to the frontend domain(s) only.

✅ FIXED
CORS restricted to: mps-frontend-qa-app.vercel.app, mps-staging.vercel.app, localhost:3000
SEC-6

No rate limiting

5 rapid sequential requests all returned 200 OK. No throttling or X-RateLimit headers. Vulnerable to scraping and denial-of-service.

✅ FIXED
Rate limiting added: 50 requests/sec, 100 burst limit via API Gateway throttling.

PEND-1

Role-based authorization for admin-only endpoints

Currently JWT authentication is enforced, but any authenticated user can access admin endpoints like POST /users, DELETE /users/{id}, and audit log writes. Need to check user's Cognito groups/role before allowing access.

Implementation
Add role check in Lambda handlers: extract cognito:groups from JWT claims, verify user has admin or manager role before processing admin-only requests. Return 403 Forbidden if unauthorized.
✅ FIXED
Role-based authorization implemented in user-manager and audit-logger Lambdas:
  • GET /users (list) — requires admin/manager
  • POST /users — requires admin/manager; only admin can create admin users
  • PUT /users/{id} — requires admin/manager; only admin can modify admin users or assign admin role
  • DELETE /users/{id} — requires admin only
  • PUT /roles — requires admin only
  • POST /audit — requires admin/manager
Unauthorized requests return 403 Forbidden with INSUFFICIENT_PERMISSIONS code.

PEND-3

Server-side filtering (multi-value filters, search)

Currently filtering is done client-side. For large datasets, need server-side filtering with support for multi-value filters (e.g., ?siteIds=1,2,3) and search across fields.

Implementation
Add query parameters to list endpoints: ?search=term (searches name, description fields), ?siteIds=1,2,3 (comma-separated IDs), ?status=active,pending (multi-value enums). Build dynamic SQL WHERE clauses with parameterized queries.
✅ FIXED
Multi-value filtering implemented in reference-data and user-manager Lambdas. All list endpoints now support:
  • ?search=term — Text search across relevant fields
  • ?ids=id1,id2,id3 — Multi-value ID filter
  • ?status=active,pending — Multi-value status filter
  • ?customerIds=c1,c2 — Multi-value customer filter (sites, profiles)
  • ?states=MI,OH,IN — Multi-value state filter (sites, facilities, transporters)
  • ?roles=admin,manager — Multi-value role filter (users)
  • ?categories=hazardous,non-hazardous — Multi-value category filter (waste types)
Uses parameterized SQL queries to prevent SQL injection.
