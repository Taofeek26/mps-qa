Security Audit

Critical

Zero Authentication Enforcement

Every API endpoint accepts requests without any Authorization header. All data — including user PII, shipment financials, and audit logs — is publicly readable, writable, and deletable by anyone with the API URL.

SEC-1

All endpoints accept unauthenticated requests

Tested with no auth header and fake tokens — both return 200 OK. JWT tokens are never validated. GET /users exposes all 12 user emails, Cognito sub IDs, and Entra IDs publicly.

Required Fix
Add Cognito JWT authorizer to API Gateway for all endpoints. User Pool: us-east-1_23veUlUUb
✅ FIXED
Cognito JWT authorizer added to API Gateway. All endpoints now require valid Authorization header. No scopes required (supports federated/Microsoft SSO tokens).
SEC-2

Anyone can create, update, and delete any record

Unauthenticated POST, PUT, DELETE all succeed. Tested: customer creation, shipment deletion, user creation with "role":"admin" (which also triggers an invitation email).

Required Fix
Add role-based authorization. Admin-only: user management, audit-log writes, system configuration.
✅ FIXED
JWT authentication now required for all write operations. Unauthenticated requests return 401.
SEC-3

Privilege escalation via unauthenticated user creation

curl -X POST -H "Content-Type: application/json" \
  -d '{"email":"attacker@evil.com","name":"Attacker","role":"admin"}' \
  .../prod/users
# Response: 201 Created — "User created. Invitation email sent."

Creates a user with full admin permissions and sends an invitation email to an arbitrary address.

SEC-5

CORS allows all origins (Access-Control-Allow-Origin: *)

Any website can make cross-origin API calls. Should be restricted to the frontend domain(s) only.

✅ FIXED
CORS restricted to: mps-frontend-qa-app.vercel.app, mps-staging.vercel.app, localhost:3000
SEC-6

No rate limiting

5 rapid sequential requests all returned 200 OK. No throttling or X-RateLimit headers. Vulnerable to scraping and denial-of-service.

✅ FIXED
Rate limiting added: 50 requests/sec, 100 burst limit via API Gateway throttling.
